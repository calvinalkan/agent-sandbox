# Documentation: README and Usage Examples

## Background & Context

A good README is essential for adoption. It should cover:
- What the tool does
- How to install it
- Basic usage examples
- Configuration reference
- Common use cases

## Rationale

The SPEC.md and TECHNICAL_STEERING.md are internal documents. Users need a
friendly, example-driven README that gets them started quickly.

## Implementation Details

README structure:
1. **Introduction** - What is agent-sandbox, why use it
2. **Installation** - How to install (binary, from source)
3. **Quick Start** - Simplest usage example
4. **Configuration** - Config file format, presets, CLI flags
5. **Command Wrappers** - @git preset, custom wrappers
6. **Examples** - Common scenarios with complete configs
7. **Troubleshooting** - Common issues and solutions
8. **FAQ** - Frequently asked questions

**Key examples to include:**
```bash
# Basic usage
agent-sandbox npm install

# Disable network
agent-sandbox --network=false npm test

# Protect additional paths
agent-sandbox --ro src/auth/ node build.js

# Project config
cat .agent-sandbox.jsonc

# Global config
cat ~/.config/agent-sandbox/config.jsonc
```

## Files to Create/Modify
- README.md - Main documentation

## Key Invariants
- All examples are tested and work
- Config examples are valid JSON/JSONC
- Installation instructions work on fresh system
- Links to SPEC.md for detailed reference

## Acceptance Criteria

1. README has introduction explaining purpose
2. Installation instructions for binary and from-source
3. Quick start with simplest working example
4. Configuration section with all options
5. Examples for common use cases
6. Troubleshooting section
7. All examples tested and working
8. Proper markdown formatting
